# HDMI CEC — Specifications

| Req ID | Test Scope |
|:-------|:--------------------------------------|
| `VTS‑HDMICEC‑001` | SHALL open the CEC HAL and obtain a valid non-zero handle via `HdmiCecOpen()` returning `HDMI_CEC_IO_SUCCESS`, and close the handle via `HdmiCecClose()` returning `HDMI_CEC_IO_SUCCESS`, supporting a subsequent re-open after close. |
| `VTS‑HDMICEC‑002` | SHALL retrieve the device physical address via `HdmiCecGetPhysicalAddress()` returning `HDMI_CEC_IO_SUCCESS` with a valid physical address value reflecting the HDMI topology. |
| `VTS‑HDMICEC‑003` | SHALL retrieve the HAL-allocated logical address via `HdmiCecGetLogicalAddress()` returning `HDMI_CEC_IO_SUCCESS` with a valid logical address value, and, when the source device is not connected to a sink, report logical-address unavailability by returning `HDMI_CEC_IO_LOGICALADDRESS_UNAVAILABLE` from `HdmiCecOpen()`. |
| `VTS‑HDMICEC‑004` | SHALL register a receive callback via `HdmiCecSetRxCallback()` returning `HDMI_CEC_IO_SUCCESS`, including acceptance of a NULL callback to deregister. |
| `VTS‑HDMICEC‑005` | SHALL transmit a CEC frame synchronously via `HdmiCecTx()` returning `HDMI_CEC_IO_SUCCESS`; for a frame addressed to a non-existent device the transmission is reported as sent but not acknowledged (`HDMI_CEC_IO_SENT_BUT_NOT_ACKD`), and a broadcast frame returns `HDMI_CEC_IO_SUCCESS`. |
| `VTS‑HDMICEC‑006` | SHALL, as a source device, report the sink-only logical-address management operations `HdmiCecAddLogicalAddress()` and `HdmiCecRemoveLogicalAddress()` as unsupported by returning `HDMI_CEC_IO_OPERATION_NOT_SUPPORTED` for all invocation scenarios. |
| `VTS‑HDMICEC‑007` | SHALL enforce the following error code contracts across all `HDMICEC` APIs:<br>`HdmiCecOpen()` returns `HDMI_CEC_IO_ALREADY_OPEN` when it is called while the CEC HAL is already open<br>every `HDMICEC` API returns `HDMI_CEC_IO_NOT_OPENED` when it is invoked without a prior open or after the handle has been closed<br>every `HDMICEC` API returns `HDMI_CEC_IO_INVALID_HANDLE` when called with an invalid handle, and `HDMI_CEC_IO_INVALID_ARGUMENT` when called with a NULL handle, data, or output pointer or an invalid data length. |
