# RDK-V Guided Test Suite — Specifications

| Req ID | Test Scope |
|--------|------------|
| `GT-REQ-001` | SHALL support complete Bluetooth A2DP audio device connection lifecycle — including device discovery, pairing, connection, disconnection, unpairing, and automatic reconnection after device power cycle — with each state correctly reflected by the Bluetooth device management subsystem on the DUT. |
| `GT-REQ-002` | SHALL route active A2DP audio output to a paired external Bluetooth audio device during media playback, and correctly apply volume level adjustments and mute/unmute transitions — with each state audibly confirmed on the connected device. |
| `GT-REQ-003` | SHALL expose complete and accurate device metadata — including name, MAC address, manufacturer ID, device type, and battery level — for a paired and connected external Bluetooth A2DP audio device through the Bluetooth device information subsystem. |
| `GT-REQ-004` | SHALL correctly detect HDMI hotplug events and manage HDCP connection state transitions — updating disconnection state on HDMI cable removal and triggering authentication initiation on HDMI reconnect — through the HDCP content protection subsystem. |
| `GT-REQ-005` | SHALL complete full HDCP authentication between the DUT and the connected HDMI display during active AV media playback — confirming authentication, enabled, and compliance status — with no degradation in audio or video quality during streaming. |
| `GT-REQ-006` | SHALL accurately report HDCP protocol hardware support and consistent HDCP 2.2 version negotiation — including supported, received, and current version parameters — through the HDCP content protection subsystem during active AV playback. |
| `GT-REQ-007` | SHALL decode and render PNG, JPEG, SVG, and WebP image formats correctly via the RDK platform browser engine, with each image format loaded and visually confirmed as displayed correctly on the connected TV display. |
| `GT-REQ-008` | SHALL correctly report IPv6 internet connectivity status and retrieve valid public IPv6 addresses for both Ethernet and WiFi interfaces through the RDK network management subsystem, across dual-network and WiFi-only configurations. |
| `GT-REQ-009` | SHALL provide functional IPv6 internet access and support uninterrupted AV media playback via both the Ethernet interface in a dual-network configuration and the WiFi interface in a WiFi-only configuration. |
| `GT-REQ-010` | SHALL support IPv6 network diagnostic capabilities — including ICMP ping reachability, route tracing, and public IP address resolution with interface-specific query support — through the RDK network management subsystem. |
| `GT-REQ-011` | SHALL correctly transition to LIGHT SLEEP power state and subsequently wake up to the ON state with functional AV playback restored, completing the full LIGHT SLEEP power management cycle without failure. |
| `GT-REQ-012` | SHALL maintain the SSH remote access service in active running state with valid RDK firmware identity confirmed, and maintain the complete WPEFramework middleware process group in actively running state on the DUT. |
| `GT-REQ-013` | SHALL execute the log rollover mechanism correctly during device reboot cycles, producing previous session log files in the designated rollover directory — confirming the RDK log management subsystem correctly rotates log data essential for post-failure diagnostics and system auditability. |
| `GT-REQ-014` | SHALL correctly instantiate and manage the Web Audio API AudioContext lifecycle — including creation with expected confirmation and repeated create/destroy cycles — without crash, memory leak, or unrecoverable error. |
| `GT-REQ-015` | SHALL support Web Audio API audio synthesis and playback capabilities — including audio node playback, FM sound synthesis, and concurrent multi-stream playback of video, audio, and speech synthesis — without conflict or failure. |
| `GT-REQ-016` | SHALL produce audible text-to-speech output via the Web Audio Speech Synthesis API for basic speech synthesis, supported audio language voice profiles, and continuous multi-language synthesis using randomly selected voice profiles — with all variants audibly confirmed without errors. |
| `GT-REQ-017` | SHALL decode AAC, MP3, and Vorbis OGG compressed audio formats at multiple bitrates via the Web Audio API, with decoded codec information correctly processed and displayed by the RDK platform browser. |
| `GT-REQ-018` | SHALL decode WAV audio at 24-bit depth at both standard 44kHz and non-standard 22kHz (requiring resampling) sample rates via the Web Audio API, with correct codec information processed and displayed by the RDK platform browser. |
| `GT-REQ-019` | SHALL complete the XCONF-based firmware upgrade workflow — including firmware download from the configured XCONF server, flash completion, and upgrade success confirmation — meeting XCONF image upgrade certification requirements. |
