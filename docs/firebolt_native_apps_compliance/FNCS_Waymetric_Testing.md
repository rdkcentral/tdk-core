# FNCS_Waymetric_Testing Test Case Documentation

## TestCase ID
FNCS_GRAPHICS_04

## TestCase Name
FNCS_Waymetric_Testing

## Objective
To execute the waymetric application and validate performance metrics that measure the impact of Wayland display server behavior versus direct EGL rendering on graphics performance.

## Preconditions

| ID | Conditions |
|----|------------|
| 1 | TDK_FNCS_Package should be installed in DUT |
| 2 | waymetric binary should be available in the device |
| 3 | RunGraphicsTDKTest.sh script should be available in /opt/TDK directory on DUT |
| 4 | DUT must support running the waymetric graphics performance application |
| 5 | wpeframework should be stoppable if running to obtain EGL display for the application |
| 6 | Command `cd /opt/TDK ; sh RunGraphicsTDKTest.sh Waymetric` must be executable on DUT |

## Test Steps

| ID | StepName | Step Description | Expected Result |
|----|----------|------------------|----------------|
| 1 | Initialize Graphics Test Environment | Setup environment for graphics validation using platform specific environment variables from TDK.env. Stop wpeframework if required to obtain EGL display. Ensure the display environment is ready for performance benchmarking. | All environment variables must be set successfully, wpeframework must be stopped if required, and the display environment must be ready to execute the waymetric application. |
| 2 | Validate Test Artifacts | Confirm that waymetric binary and RunGraphicsTDKTest.sh are present and executable on the DUT. | waymetric binary and RunGraphicsTDKTest.sh must be available and executable in expected DUT paths. |
| 3 | Execute Waymetric Performance Test | Run the waymetric performance test command on DUT using `cd /opt/TDK ; sh RunGraphicsTDKTest.sh Waymetric`. | Environment variables must be set successfully, waymetric binary must be accessible, the command must be constructed properly with correct parameters, and the test application must start execution without errors. |
| 4 | Collect Performance Metrics | Capture the waymetric application output and the speed indices generated by the test. | waymetric output must be captured successfully and speed indices must be available for analysis. |
| 5 | Verify Output Integrity | Validate that the waymetric output does not contain error statements and that the reported metrics are complete. | Output must not contain `error` or `ERROR` statements and speed index results must be present. |
| 6 | Determine Test Result | Based on command execution and output verification, determine whether the Waymetric performance test passes. | Test result must be marked SUCCESS if the command executed without errors and speed indices were successfully obtained; otherwise mark FAILURE. |
