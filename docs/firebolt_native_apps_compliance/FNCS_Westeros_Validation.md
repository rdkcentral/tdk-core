# FNCS_Westeros_Validation Test Case Documentation

## TestCase ID
FNCS_GRAPHICS_03

## TestCase Name
FNCS_Westeros_Validation

## Objective
To execute the Westeros_TDKTestApp for EGL and Westeros renderer validation, confirming that the graphics application launches correctly, initializes the compositor path successfully, and produces output without error statements.

## Preconditions

| ID | Conditions |
|----|------------|
| 1 | TDK_FNCS_Package should be installed in DUT |
| 2 | Westeros_TDKTestApp test binary should be available in the device |
| 3 | RunGraphicsTDKTest.sh script should be available in /opt/TDK directory on DUT |
| 4 | DUT must support Westeros compositing with EGL display access |
| 5 | Display environment must allow the Westeros renderer to create and use a display named `wayland-0` or equivalent |
| 6 | Command `cd /opt/TDK ; sh RunGraphicsTDKTest.sh Westeros 30` must be executable on DUT |

## Test Steps

| ID | StepName | Step Description | Expected Result |
|----|----------|------------------|----------------|
| 1 | Initialize Graphics Test Environment | Setup environment for graphics validation using platform specific environment variables from TDK.env. Ensure the display compositor or Westeros renderer backend is configured for EGL validation. | All environment variables must be set successfully and the display environment must be ready for Westeros application launch. |
| 2 | Validate Test Artifacts | Confirm that Westeros_TDKTestApp and RunGraphicsTDKTest.sh are present and executable on the DUT. | Westeros_TDKTestApp binary and RunGraphicsTDKTest.sh must be available and executable in expected DUT paths. |
| 3 | Execute Westeros Graphics Validation | Run the Westeros graphics test command on DUT using `cd /opt/TDK ; sh RunGraphicsTDKTest.sh Westeros 30`. | Environment variables must be set successfully, the Westeros application binary must be accessible, the command must be constructed properly with correct parameters, and the test application must start execution without errors. |
| 4 | Initialize Westeros Graphics Context | Westeros_TDKTestApp initializes the EGL display and configures the Westeros renderer for the selected display backend. | EGL display and Westeros renderer must initialize successfully and the application must connect to the compositor path. |
| 5 | Validate Renderer Output | Verify that the Westeros application produces validation output without error statements and that the renderer flow completes as expected. | Application output must not contain error statements and must indicate successful EGL/Westeros validation. |
| 6 | Capture Application Output | Collect the full output generated by Westeros_TDKTestApp during test execution. | Output must be captured completely and available for parsing. |
| 7 | Parse Graphics Validation Output | Parse captured output to verify absence of `error` statements and confirm that no validation failures are reported. | Output must not contain `error` statements and validation success indicators must be present. |
| 8 | Determine Test Result | Based on command execution, output capture, and output parsing, determine whether the Westeros validation test passed. | Test result must be marked SUCCESS if the command executed without errors and the output indicates no validation failures; otherwise mark FAILURE. |
